使用hexo搭建github博客（基于win7）

hexo是一个台湾人基于Node.js开发的静态博客程序，优势是生成静态文件的速度非常快，支持Markdown。最初，我选定pelican-python作为静态博客的框架，但win7配置起来确实太折磨人了。这里有人使用hexo部署静态博客，而且是基于win7 64系统的，我也马上尝试了一下，发现真的很简单。现将安装、部署记录下来，以便备用。

安装

这是基于win7 64位系统，Hexo是在淘宝镜像下载最新的版本。

首先，进入node.js镜像，下载最新版本。然后，按照淘宝镜像的使用说明，使用cnpm命令行工具代替默认的npm

npm install -g cnpm –registry=https://registry.npm.taobao.org

以上命令在命令行窗口执行即可（win+R）

安装依赖软件

  • Node.js已说明
  • Git客户端（百度收索，不论是哪个版本都可以的）

安装hexo

利用cnpm命令安装，在窗口控制台，输入以下命令

cnpm install -g hexo

初始化

安装完成后，cd到你将要放置hexo文件的文件夹，执行如下命令

hexo init

安装依赖包

cnpm install

然后生成和发布命令如下。我们可以等到配置完成了再执行以下命令

hexo g #其实是 hexo generate命令
hexo s #其实是 hexo server命令

然后访问浏览器端口http://localhost:4000，即可看到生成的博客

发布到github

首先注册github账号。记住github username将是最终生成的博客地址的前缀。

创建repository

登陆github，进入个人页面，在+号下选择new repository。创建repository时，将Repository Name命名为yourname.github.io的格式。当然yourname是你注册github时使用的username。使用其他名字命名的使用静态网无法访问。

更改本地配置文件

在hexo文件夹里找到_config.yml文件，并添加以下内容。记住，hexo配置文件中任何一个:后面必须带一个空格。

deploy:
type: git
repository: https://github.com/yidao620c/yourname.github.io.git
branch: master

配置ssh。当然，不懂的可以直接删除ssh配置，并重新设置。

在hexo文件夹，右键，点击Git Bash打开git命令窗口，之后的命令都可以在git bash命令窗口中执行(实际上git bash有些类似win命令窗口)

使用如下查看是否存在ssh配置

ls -al ~/.ssh

进入/administrator/.ssh删除该文件夹里的内容。

输入指令，新建ssh。记住将邮箱修改为github注册邮箱

ssh-keygen -t rsa -C “youremail@gmail.com”
eval ssh-agent -s
ssh-add

拷贝ssh key到剪切板

clip < ~/.ssh/id_rsa.pub

然后到github，setting—ssh keys—new—输入title(随意)—ctrl+v粘贴key—Add SSH Key—输入密码确认。OK

测试。

ssh -T git@github.com

看到警告后，输入”yes”，可以看到提示连接成功。

安装必要的模块

cnpm install hexo-deployer-git –save

SSH配置完成，执行以下命令部署到github

hexo g
hexo d

提示输入github账号和密码，就能访问博客网站了。注意，第一次输入的是您的username第二次输入的是密码。部署成功以后，在浏览器地址栏输入http://yourname.github.io/即可访问网页。

如何发表一篇博客

在hexo文件夹，控制台执行命令(git bash也可以)

hexo new “Hello World”

在hexo\source\_posts文件夹中生成了hello-world.md文件，用编辑器打开，并开始编辑文字吧。hexo遵循markdown语法。里面的初始内容是

title: hello world # 可以改成中文的
date: 2016-6-1 16:04:09 # 发表日期，一般不改动
tags: blog # 文章分类

#这里是正文，用markdown写，可以选择写一段显示在首页的简介，并在后面加上<!--more-->。表示之后的内容被隐藏了，需要点击read more查看
写完文章后，使用hexo g生成静态文件，hexo s在本地预览，hexo d同步到github。

hexo主题及配置

如果不喜欢默认主题，可以到hexo官网找更多主题。此处以maupassant-hexo主题为例

安装主题

$ git clone https://github.com/tufu9441/maupassant-hexo.git themes/maupassant
$ cnpm install hexo-renderer-jade –save
$ cnpm install hexo-renderer-sass –save

一般来说，github的主题会介绍安装步骤

启用主题

修改_config.yml配置文件中的theme属性，将其设置为maupassant

更新主题

cd themes/maupassant
git pull

更新好以后，退回到hexo文件夹，本地启动效果

hexo server -g #生成加预览

主题的_config.yml配置

修改相关细节，使它更符合自己的使用习惯。

维护

维护的中心思想是新建一个分支，然后通过hexo维护master分支，通过git命令维护新建的分支。在另一台电脑使用时，先切换到新建的分支，而不使master受到影响。

具体步骤见原文

自定义分页

安装三个插件：归档、标签、分类。

cnpm install hexo-generator-archive –save
cnpm install hexo-generator-tag –save
cnpm install hexo-generator-category –save

三个插件的配置：

archive_generator:
per_page: 100
yearly: true
monthly: true
daily: false
tag_generator:
per_page: 100
category_generator:
per_page: 100

绑定自己的域名

Github Page绑定自己域名，在source/目录下面新建CNAME文件，里面写入你自己的域名比如

www.pycoding.com

去自己的域名运营商处添加CNAME类型的DNS记录:

CNAME: @ => yidao620c.github.io
CNAME: www => yidao620c.github.io

如果是DNSPod，那么后面多加个点

@，CNAME，cmback.github.io.
www，CNAME，cmback.github.io.

设定tags

  1. 新建一个页面，命名为tags。

hexo new page “tags”

  1. 编辑新建的页面，配置如下

title: Tagcloud
date: #不要更改
type: “tags”
comments: false

  1. 添加主题链接。编辑主题配置文件，添加tags到menu。

总结

  • cd hexo部署文件夹
  • hexo new “postname” # 新建博文
  • cd source/_posts修改编辑
  • hexo generate # 生成或修改之后都需要重新运行一次
  • hexo server # 发布
  • 或者 hexo server -g # 先生成后预览
  • 本地浏览器地址http://localhost:4000
  • hexo deploy # 或者hexo d 上传到github
  • 输入username和code
  • http://yourname.github.io/访问
