rpy2

关于数据分析是学习Python和R的争论由来已久，但并未涉及问题的本质。实际上，任何一个工具，若能精通，都能完成我们想要完成的任务，区别在于效率、实现的成本(包括无形的、有形的)、是否感觉很酷以及其他方面。我也在两者之间有过徘徊，但并没有耗费太多时间，就选择将两者结合起来。当然，这与我的专业有关。我接触编程语言不是半路出家，而是中间中断好几年。这些暂且不说，我认为两种语言，如编程语言，各有特色。

R：大量的、由专业统计人员编写的package。这部分package，如关在笼子里的猛兽，并没有充分发挥它的作用。缺点是运算速率不尽人意。近年来，Julia逐渐在经济学家中流行起来，也说明了在运算效率上，R已经让一些具有编程能力的学者难以接受，而寻找其他替代方案。

Python：动态编程语言。具有很强的灵活性，语法简洁、代码易于维护，而且其胶水语言的特性使得它很容易与其他语言结合起来，从而很容易实现拿来主义。这一点，我认为，使Python比其他编程语言更有优势。一方面，Python能通过简单的API使用其他语言的库；另一方面，其简洁的语法也使得其他平台的优秀库很容易的移植到Python平台。如R中的ggplot2在python中已经有对应的package，matlab的很多功能在python中都能找到对应的package。缺点是不够专业。毕竟程序员与从事专门学科研究的人相比，在问题的理解上存在很大差别。有人会说Python的语法并不难以掌握，但对于编写高效的代码，还是存在很多障碍。Python的运行效率很大程度上取决于使用的方法或是否使用package。两者的运行效率可能相差几个数量级。如果不理解这些，在Python中使用了循环，那么运行速率和R相比就没有优势了。

安装

安装步骤：MRO—配置R—Anaconda—rpy2

R：

Microsoft R Open 3 64-bit

Python：

Anaconda 64-bit

rpy2：

rpy2.whl

解释：

  • MRO是微软收购Revolution R Open以后更名的R安装包，增加了多进程并行计算，对多核计算机R计算有一定的性能提升。另外，推荐安装MRO对应的数学计算库Math Library提高运行效率。安装步骤是先安装MRO，再安装Math Library。两个都是可执行文件，Math Library会自动寻找到MRO安装文件夹进行安装
  • Anaconda是python安装包，一步解决所有包安装问题
  • rpy2是伯克利大学制作的一个win系统下的python安装包，省去了调试、链接等繁琐的步骤。一般来说，.whl文件包含了安装过程中所需要的链接文件。安装是cd到下载文件，使用pip install rpy2.whl

R配置：

  • 添加path：path: D:\...\MRO\R\bin\x64
  • 添加R.dll到path:path: D:\...\MRO\R\bin\x64\R.dll
  • 设置R_HOME环境变量：新建，R_HOME，D:...\MRO\R
  • 添加path: %R_HOME%，注意，需要将它移到path的最前面。
  • 设置R_USER：新建，R_USER，Administrator。可以通过cmd—net user查看用户名。

调用

python导入模块

import rpy2.robjects as robjects

三种方法调用R

  • robjects.r[‘pi’]
  • robjects.r(‘pi’)
  • robjects.r.pi

一般来说，第二种方法是通用的。可以将任意大小和长度的R代码写成一个python字符串，然后通过robjects.r(‘R_code_string’)调用执行。

第三种方法对于名称中有点号的变量会出问题。

Advanced R

data structure

R语言的基础数据结构(几乎所有的其他对象都是建立在以下五类基础数据结构之上)

Homogeneous Heterogenous
1d Atomic vector List
2d Matrix Data frame
nd Array

R语言没有0维数据结构（标量），单个数字或字符串都是长度为1的向量。使用str()函数(structure缩写)查看任何R语言数据类型。

R语言中最基本的数据结构是向量，包括Atomic和List，具有以下属性：

  • typeof()：类型
  • length()：长度
  • attributes()：属性

Note：is.vector()不能测试对象是否是向量，只有当对象是除了名字以外，不包含其他属性时，该函数可用。需判断是否向量，使用is.atomic(x)||is.list(x)测试。

1d

atomic vector

四种常见的atomic vector：逻辑类型、整数、双精度（又称数值类型）和字符类型。罕见的，复数类型和raw类型。

atomic vector由c()创建，combine，因此c()也能用来合并几个向量，合并前c()会将atomic vector强制转换成list.

  • 使用L后缀，得到整数，默认双精度
  • 使用TRUE和FALSE或(T/F)创建逻辑向量
  • 使用双引号创建字符串

atomic vector总是平的(只有一维嘛)，c(1,c(2,c(3,4)))与c(1,2,3,4)相同。

缺失值NA，是长度为1的逻辑向量。c()函数总是强制将NA转换为正确的类型，或者使用NA_real_（双精度）、NA_integer_（整数）、NA_character_来指定。

typeof()与is.character()、is.double()、is.integer()、is.logical()甚至是is.atomic()通用。

NOTE.1:is.numeric()测试向量而非双精度，虽然双精度称为数值(numeric)。整数、字符串、双精度返回都是TRUE。

NOTE.2:强制转换向最灵活的类型进行。灵活性从小到大依次为逻辑、整数、双精度、字符串。

NOTE.3:as.numeric()、as.character()、as.double()、as.integer()、as.logical()强制类型转换。

List

list元素可不同，甚至是list。创建使用list()函数。list有时被称为递归向量，因为list可以包含其他list，使用is.recursive(x)判断。

is.list()和as.list()、unlist()测试、强制转换list，以及将list转为atomic vector(如果类型不同，则不行)。

list用来创建更加复杂的数据结构，如数据框、线性模型对象(lm()产生)。

属性

所有对象都可以拥有任意多个附加属性，附加属性用来存取与该对象相关的元数据。属性可看作是已命名的list(带有不重复的名字)。属性可以使用attr()函数逐个访问，也可以使用attributes()函数一次性访问。

NOTE.1: 修改属性attr(x,"my_attr_name")<-"this..."，str(attributies(x))则返回的是一个list.

NOTE.2: 默认情况下，修改向量(atomic vector,list)，大多数属性会丢失。但是有三个重要的属性不会丢失。

这三个属性：

  • name: 字符串向量
  • dimension: 维度
  • class：类

这些属性都有一个特定的访问函数，names(x)、class(x)、dim(x)，而不是attr(x,"names")…。(names说明向量可以对每个元素命名，使用names(x)<-c(“a1”,”a2”,…)或使用setNames(1:3,c(“a1”,”a2”,”a3”))使用后者对前者命名。当然，也并不需要都命名，names()函数将使用空字符串，即""，如果所有的元素都无名，则返回NULL。因此，也可以使用unname(x)或names(x)<-NULL创建无名向量。)

因子

属性的运用。仅包含预定义值，用来保存category或level，相当于枚举，构建于整数向量之上。

factor()带有两个属性：

  • class()，区别于普通整数向量
  • levels()，定义了允许取值集合。
